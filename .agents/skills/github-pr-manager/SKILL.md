## Instructions

1. **Status Check:** Run `git status` and `git diff` to analyze the changes made to the codebase.
2. **Staging:** Stage all modified files using `git add .`.
3. **Commitment:** - Generate a professional, concise commit message using the "Conventional Commits" standard (e.g., `feat:`, `fix:`, `refactor:`).
   - Execute `git commit -m "<message>"`.
4. **Integration:** - Push the current branch to the remote `origin`(create if missing).
   - Use the **GitHub CLI** (`gh pr create`) to open a Pull Request.
   - **PR Title:** Use the generated commit message.
   - **PR Body:** State that this PR was automatically generated by Codex during a technical demonstration.

## Constraints

- **Safety:** If `git status` shows no changes, inform the user and abort the skill.
- **Environment:** Always use the `gh` command-line tool for PR creation to ensure integration with the GitHub ecosystem.
